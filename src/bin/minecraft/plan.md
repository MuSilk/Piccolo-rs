# Minecraft 原型推进计划

本文档按「已具备能力 → 近期可交付 → 中期扩展」分解任务，便于按里程碑实现与验收。

---

## 0. 当前基线（已实现）

- **世界**：固定 `12×12` 区块阵列；每块 `16×16×256`；噪声高度场 + 草/土/岩分层。
- **渲染**：每区块一个 `MeshComponent` + `GameObjectDynamicMeshDesc`，`Chunk::update_mesh_data` 生成面（邻格为空气才画面）。
- **玩家**：`Character` + `Camera` + `Motor` + `CharacterController`，与方块 AABB 做分轴滑动碰撞（`World::get_aabbs`）。
- **方块类型**：空气、土、草、石；纹理来自图集 UV（`block.rs` + `block.json` 资源）。

---

## 1. 近期：正确性与体验（建议优先顺序）

### 1.1 区块边界接缝

- **问题**：`Chunk::update_mesh_data` 在边界外一律「视为实心」并保留外侧面，相邻区块接缝处会重复面或视觉不连续（取决于期望）。
- **任务**：
  - 在 `World` 上提供 `get_block_world(wx, wy, wz) -> BlockType`（或 `Option`），能跨区块查邻居。
  - `update_mesh_data` 改为接受「查邻居」闭包或 `&World` + 本块 `(chunk_i, chunk_j)`，边界处用真实邻居类型决定是否建面。
- **验收**：相邻两区块交界处无多余内壁、无错误镂空。

### 1.2 世界坐标与变换一致性

- **问题**：`world.rs` 里 `TransformComponent::set_position` 使用 `(i*16, j*16, 0)`，需与 `chunk` 内 `(x,y,z)` 与玩家坐标系约定一致（谁代表东/北/上）。
- **任务**：在代码或本文档中写清「世界轴含义」与「区块原点」；统一 `get_aabbs` / 网格索引 / 相机前向，必要时修正 `set_position` 或 `chunk` 顶点布局。
- **验收**：玩家沿四方向移动时与地形贴合无系统性偏移。

### 1.3 破坏与放置（最小闭环）

- **射线**：从相机位置沿前向步进或引擎已有 pick API，求第一个非空气方块及其面法线。
- **破坏**：将该格设为 `BLOCK_AIR`，标记该 `Chunk` 网格 dirty，重建/更新对应 `DynamicMesh`。
- **放置**：在命中面相邻格放置当前手持方块类型（先做单一类型即可）。
- **任务**：在 `Scene::tick` 或独立系统中读输入（鼠标键）；`World` 提供 `set_block_world` + 根据 `(chunk_i, chunk_j)` 找到对应 `GameObject`/mesh 引用并刷新。
- **验收**：可挖洞、可叠砖；碰撞体与网格同步更新。

### 1.4 碰撞与性能小优化

- **任务**：`get_aabbs` 仅遍历与 AABB 相交的整数格（已有大致循环，可收紧边界）；考虑缓存或粗检测避免每帧全扫。
- **可选**：斜坡/台阶不做，保持轴对齐即可。

---

## 2. 中期：世界规模与内容

### 2.1 动态区块加载（取代固定 12×12）

- **任务**：以玩家所在区块为中心维护 `N×N` 窗口；远离的区块卸载（销毁对象或隐藏）；进入范围则生成地形 + mesh（可复用 `World::new_box` 中单块生成逻辑）。
- **依赖**：1.1 跨块邻居查询（动态邻居更关键）。
- **验收**：向一个方向走远仍可持续，内存不无限增长。

### 2.2 存档与读档

- **任务**：实现 `Scene::save` / 加载路径：序列化每个区块 `BlockType` 网格（或 RLE）；恢复时重建 mesh。
- **验收**：退出再进游戏，修改过的地形保留。

### 2.3 更多方块与合成表（简化）

- **任务**：扩展 `BlockType` + 图集坐标；`block.json` 或数据驱动列表；手持方块切换（数字键循环）。
- **验收**：至少多 3～5 种装饰性或功能性方块。

### 2.4 昼夜与简单雾效（可选）

- **任务**：随时间调 `render_system` 环境光或雾颜色（若引擎暴露接口）。
- **验收**：肉眼可见昼夜变化即可，不求物理正确。

---

## 3. 长期：玩法与 polish（按需）

- **生物群系**：多套噪声权重切换草/沙/雪顶等。
- **水体**：单独方块类型 + 透明渲染排序（工作量大，可后置）。
- **第一人称手臂 / 方块预览**：纯表现。
- **多人**：网络同步区块 diff，单独大项。

## 4. 建议里程碑划分

| 里程碑 | 包含任务 | 产出 |
|--------|----------|------|
| M1 接缝 + 坐标 | 1.1、1.2 | 大世界视觉与碰撞一致 |
| M2 交互 | 1.3、1.4 | 可玩「挖与放」 |
| M3 无限感 | 2.1、2.2 | 可走远 + 可存档 |
| M4 内容 | 2.3、2.4 | 更像「游戏」 |

---

## 5. 技术债备忘（实现时留意）

- `Chunk::update_mesh_data` 每帧全量重建会卡：后续改为「仅 dirty 区域重建」或后台线程生成顶点再上传 GPU。
- `World` 持有 `Rc<Block>` 与 `Chunk` 内数组：修改单格时注意 `Rc` 共享语义（若多引用同一块类型则无问题；若需唯一实例再评估）。
- `CharacterController` 分轴滑动在台阶处可能「卡脚」，可后续加「上抬一步」或胶囊体近似。
